ARIA나 [접근 가능한 리치 인터냇 어플리케이션](http://www.w3.org/WAI/intro/aria.php)는 접근 가능한 유저 인터랙션을 개발하는데 도움이 되는 역할, 속성, 지침의
프레임워크를 제공합니다.

## ARIA

이 연습을 위해, 랜드마크 영역이라고 불리는 WAI-ARIA의 한 작은 부분에 초점을 맞출 것입니다. 랜드마크 영역은 스크린리더 사용자가 페이지를 훑어 볼 때 다른 사람들이 훑어 볼 때 수행하는 작업 즉, 조감도로 다른 부분을 보는 것에 도움이 됩니다. 또한 스크린리더의 읽기 커서를 페이지의 다른 위치로 쉽게 이동할 수 있게 합니다.

1. 보이스오버를 켜세요.
2. `Ctrl` + `Opt` + `u`를 누르세요.
3. “Landmarks”가 들릴 때 까지 오른쪽 방향키를 누르세요.
4. 랜드마크 목록을 읽으려면 아래 방향키를 누르세요.
5. 보이스오버 읽기 커서를 랜드마크 영역 중 하나의 시작 위치로 이동시키려면 enter를 누르세요.

import { Editor } from "../_util/Editor.mdx";

<Editor
  checkExercise={() => {
    const { value } = document.getElementById("landmarks");
    const isValid = value === "4" || value === "four";
    if (!isValid) {
      return "That is not the right number of landmarks on the page!";
    }
  }}
>
  {`<h3>ARIA Landmarks Exercise</h3>
How many landmarks did you find on the page?
<input type="text" id="landmarks" />`}
</Editor>

랜드마크 역할의 몇 가지 유형이 있습니다. 그 중 일부는 다음과 같습니다:

- `<main>`, `<div role="main">` – 스크린리더 사용자의 "시선 유도(directing the gaze)"와 동등합니다. 페이지의 주요 콘텐츠만 포함해야 합니다.

- `<nav>`, `<div role="navigation">` – 사이트 탐색을 위한 링크나 엘리먼트 모음을 포함합니다.

- `<aside>`, `<div role="complementary">` - 주요 콘텐츠를 보완하는 콘텐츠

- `<div role="search">` - 검색 위젯

- `<header>`, `<div role="banner">` - 페이지의 배너

- `<footer>`, `<div role="contentinfo">` - 바닥글에 나타나는 저작권, 도움말, 정보 링크와 같은 페이지에 대한 정보

:::tip 역자 주

여기서 "배너"는 광고 배너와 같은 배너가 아니라 웹사이트 내의 사이트 지향적인(site-oriented) 콘텐츠로, 일반적으로 로고나 사이트 제공자의 신분 같은 것들과, 사이트 지향적인 검색 도구를 포함하며  보통 페이지의 맨 위에 나타나고 일반적으로 전체 너비에 걸쳐 있습니다.

:::
